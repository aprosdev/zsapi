Customers
The people who use the mobile app

Customers / sign_in

POST /api/:version/customers/sign_in

Signs the customer in by updating the social_token and social_type

Examples

200 OK

Params

  • customer : required [

    Must be a Hash

    ]
    • email : required [

      Must be String

      ]

      Email address of customer

    • social_token : required [

      Must be String

      ]

      The social token returns by Facebook/Google+

    • social_type : required [

      Must be one of: facebook, google+.

      ]

      The type of social account connected

    • social_id : required [

      Must be String

      ]

      The social ID

    • first_name : required [

      Must be String

      ]

      Customer's first name

    • last_name : required [

      Must be String

      ]

      Customer's last name

    • social_friends : optional [

      Must be an array of any type

      ]

      An array of the customer's social friends. Example: [1,4,5,6]

Locations
The individual locations of a business

Locations / show

GET /api/:version/beacons/:uuid/location

GET /api/:version/locations/:id

Returns a location object and includes information relevant to the current customer.

There are two ways to retreive data for a single location, by inputting the beacon's UUID, or by inputting the location's ID. Use the proper route depending on what you're trying to do.

Examples

{
  "count" => 0,
  "response" => Array[
    Hash {
      "id" => ,
      "business" => Hash { Business Object },
      "title" => ,
      "address" => ,
      "address2" => ,
      "city" => ,
      "state" => ,
      "zipcode" => ,
      "latitude" => ,
      "longitude" => ,
      "status" => ,
      "points" => , 	# The current customer's points for this location
      "visits" =>   	# The number of checkins for the current customer at this location
    },
    # ...
  ]
}

Params

  • id : optional [

    Must be a number.

    ]

    The location ID. Required if: Using the ID route

  • uuid : optional [

    Must be a number.

    ]

    The beacon's UUID. Required if: Using the UUID route
